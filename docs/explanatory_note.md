# Пояснительная записка

## Название проекта
ReshuLyceum

## Описание идеи
ReshuLyceum помогает школьникам готовиться к ЕГЭ по информатике. Проект объединяет каталог задач, автопроверку ответов, личную статистику, игровые механики и адаптивные рекомендации.

## Актуальность
При подготовке к экзамену важно не просто решать задачи, а видеть прогресс, понимать слабые темы и получать задания подходящей сложности. Приложение решает эту проблему через статистику, рейтинг и рекомендации.

## Архитектура
Проект разделён на слои:
- `app/api` содержит REST API.
- `app/web` содержит HTML-маршруты.
- `app/templates` содержит Jinja2-шаблоны.
- `app/static` содержит CSS, JavaScript и загружаемые файлы.
- `app/models` содержит ORM-модели SQLAlchemy.
- `app/schemas` содержит Pydantic-схемы.
- `app/services` содержит бизнес-логику: проверка ответов, рейтинг, достижения, уровни сложности и обновление схемы БД.
- `scripts/seed_data.py` создаёт демо-данные.
- `tests` содержит автотесты.

## База данных
Используется SQLite. Основные таблицы:
- `users` хранит пользователей, пароль в хэшированном виде, рейтинг, XP, уровень, серии и путь к аватару.
- `tasks` хранит задачи, темы, сложность, ответ и объяснение.
- `attempts` хранит попытки решения.
- `achievements` хранит достижения.
- `user_achievements` связывает пользователей и полученные достижения.

## Авторизация
Реализована регистрация и авторизация по JWT. После входа токен хранится на клиенте в `localStorage` и передаётся в защищённые API-запросы через заголовок `Authorization`.

## Автопроверка
Ответ пользователя нормализуется: убираются лишние пробелы, регистр не учитывается, десятичная запятая и точка считаются эквивалентными. После проверки создаётся запись попытки.

## Адаптивная сложность
У задач есть числовая сложность и понятный уровень: очень легко, легко, средне, сложно, очень сложно. Рекомендации выбирают задачи, близкие к рейтингу пользователя, исключая уже решённые.

## Геймификация
За правильные ответы начисляется XP. Пользователь получает уровень, рейтинг, серию дней и достижения. Достижения отображаются в профиле с иконками.

## Загрузка файлов
Пользователь может загрузить аватар в личном кабинете. Файл сохраняется в `app/static/uploads/avatars`, а путь записывается в базу данных. Допускаются JPG, PNG и WEBP до 2 МБ.

## Интерфейс
Интерфейс построен на Jinja2, CSS, Bootstrap, Font Awesome и JavaScript. В профиле используются графики Chart.js.

## Тестирование
Автотесты проверяют регистрацию, авторизацию, отправку решений, нормализацию ответов и публичный рейтинг.

## Возможности развития
- Добавить редактор задач с изменением существующих записей.
- Добавить импорт задач из файла.
- Добавить комментарии и избранные задачи.
- Развернуть проект на публичном хостинге.
