Проблема

Готовиться к ЕГЭ по информатике в одиночку — сложно:

  • Сложно подобрать задачи нужного уровня — либо слишком легко, либо слишком сложно
  • Нет обратной связи о прогрессе: что освоено, а что западает
  • Низкая мотивация для регулярных занятий
  • Готовые тренажёры либо платные, либо без аналитики

Алгоритм: адаптивная сложность (ELO)

То же, что в шахматах — рейтинг растёт и падает после каждой попытки. Покрутите ползунки:

1200
1500
Ожидаемая вероятность решения 15%
Изменение рейтинга ученика +27

Формула: E = 1 / (1 + 10^((Rtask-Ruser)/400)), Δ = K·(результат − E), K = 32. Рейтинг задачи меняется симметрично ученику.

Геймификация

XP и уровни


def xp_for_level(L):
    return int(100 * (L - 1) ** 1.5)

Нелинейный рост — следующий уровень даётся всё сложнее.

Серии дней (streaks)

  • Решил вчера и сегодня → +1 к серии
  • Пропустил день → серия сбрасывается
  • Сохраняется максимум

15 достижений

  • За количество: 1 / 10 / 25 / 50 / 100 решённых
  • За серии: 3 / 7 / 30 дней подряд
  • За уровни: 5 / 10 / 20
  • За рейтинг: 1500 / 2000
  • Снайпер: точность ≥90% при 20+ попытках
  • Эрудит: задачи по 5+ темам
  • Покоритель: решил задачу сложности 1700+

Сценарий: сдача ответа

Один POST-запрос запускает цепочку из пяти операций.

POST/api/attempts
→
checkerнормализация + сравнение
→
ratingобновление ELO
→
gamificationXP, level, streak
→
achievementsпроверка 15 условий
→
JSONрезультат + toast

Проверка ответа — допускаются:

  • Регистр и лишние пробелы (" Yes " = "yes")
  • Запятая вместо точки (3,14 = 3.14)
  • Числовая близость до 1e-6 (для float)

Безопасность

  • Пароли хэшируются bcrypt (passlib)
  • JWT с истечением 7 дней, подпись HS256
  • TaskPublic не отдаёт эталонный ответ — только после правильного решения
  • Admin-эндпоинты защищены отдельной зависимостью get_current_admin
  • Загрузка аватара ограничена: JPG / PNG / WEBP, до 2 МБ, content-type валидируется
  • Валидация всех входов через Pydantic (логин — regex, email — RFC, длины, диапазоны)
  • SQL-инъекции исключены — везде ORM, параметризованные запросы

Демонстрация

  1. Главная — заголовок, задача дня, статистика по базе
  2. Регистрация → автовход — переход в профиль
  3. Каталог задач: поиск, фильтр по теме / сложности / «нерешённые»
  4. «Рекомендовать мне» — ELO в действии
  5. Сдать неверный → верный ответ — XP, рейтинг, toast-достижение
  6. Профиль: статы, прогресс-бар уровня, два графика, история, смена пароля
  7. Лидерборд с подсветкой своей строки
  8. admin / admin123 — создание и редактирование задачи в админ-панели
  9. /docs — Swagger UI
  10. pytest -v — 16 проходящих тестов
